PM Hackathon Setup Guide

Everything you need to participate in the Microsoft PM hackathon — installed step-by-step, with a verify command after each step so you know it worked. Pick your OS and editor below; the guide will show only the steps that apply to you.

Why not just one script? Earlier versions of this repo shipped a PowerShell bootstrap. Windows Defender heuristically flags iex (irm …) download-and-execute patterns as Trojan on managed corp machines — even when the script content is clean. So we switched to plain winget / brew / apt commands you paste yourself. The old script is preserved in /legacy for advanced users on unmanaged machines.

How to use this guide: every step has a Check command first. Run it — if it prints a version, you already have that tool and can skip the install. Only run the Install command if the check fails (e.g., "command not found"). After installing, close and reopen your terminal so PATH refreshes, then re-run the check.

★ Before you start: open a terminal

All the commands below are run in a terminal / command line. Open one this way:

Windows
  • In VS Code: press Ctrl + ` (backtick, top-left of your keyboard) to open the integrated terminal. Or menu: Terminal → New Terminal.
  • In Cursor: press Ctrl + ` (backtick) — Cursor is a VS Code fork, so the same shortcut opens the integrated terminal.
  • Don't have your editor installed yet? Use one of the standalone options below — you'll install the editor later in this guide.
  • Standalone: Start menu → "Windows Terminal" (or "PowerShell" / "Command Prompt" on older Windows).
  • If you don't have Windows Terminal yet, that's fine — install it in Step 5 below. cmd or PowerShell 5.1 (built-in) work for every command in this guide.
macOS
  • In VS Code: press Ctrl + ` (backtick) to open the integrated terminal. Or menu: Terminal → New Terminal.
  • In Cursor: press Ctrl + ` (backtick) — Cursor is a VS Code fork, so the same shortcut opens the integrated terminal.
  • Don't have your editor installed yet? Use Spotlight below — you'll install the editor later in this guide.
  • Standalone: Spotlight (Cmd + Space) → type "Terminal" → Enter. Or open Applications → Utilities → Terminal.
Linux
  • In VS Code: press Ctrl + ` (backtick) to open the integrated terminal. Or menu: Terminal → New Terminal.
  • In Cursor: press Ctrl + ` (backtick) — Cursor is a VS Code fork, so the same shortcut opens the integrated terminal.
  • Don't have your editor installed yet? Use the standalone option below — you'll install the editor later in this guide.
  • Standalone: Ctrl + Alt + T on most distros, or search for "Terminal" in your app launcher.

Tip: the integrated terminal inherits your current folder as the working directory — handy once you start cloning repos.

Since you picked the CLI-only path, every command in this guide runs in your standalone terminal — no editor needed.

4. GitHub CLI (gh)

Authenticate, install gh copilot, validate Microsoft↔GitHub link.

Check (skip install if a version prints)
gh --version
Install if missing
(type -p wget >/dev/null || sudo apt install -y wget) \
  && sudo mkdir -p -m 755 /etc/apt/keyrings \
  && wget -qO- https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo tee /etc/apt/keyrings/githubcli-archive-keyring.gpg > /dev/null \
  && sudo chmod go+r /etc/apt/keyrings/githubcli-archive-keyring.gpg \
  && echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null \
  && sudo apt update \
  && sudo apt install gh -y

9. GitHub Copilot CLI

Ask Copilot for shell commands and explanations right in your terminal. Works on every OS, regardless of editor choice.

Sign in to gh first
gh auth login

Pick GitHub.com → HTTPS → browser auth. Use the GitHub account that's linked to your Microsoft work account.

Check (skip install if it prints a version)
gh copilot --version
Install the Copilot extension
gh extension install github/gh-copilot
Verify it works
gh copilot --help
gh copilot explain "git status"

First run of explain or suggest prompts you to opt into usage analytics (Y/N) — answer once and it remembers.

Common errors and fixes:
  • Error: unknown command "status" for "copilot" — gh copilot status isn't a real subcommand. Use gh copilot --help to see the valid ones (suggest, explain, alias, config).
  • Error: Unexpected problem with GitHub Copilot in the CLI on gh copilot suggest "..." — usually means your GitHub account doesn't have a Copilot license yet. Complete Step 10 below to validate the GitHub↔Microsoft link, then retry. Also try gh auth refresh -h github.com -s copilot to refresh your token scopes.

Validate your GitHub ↔ Microsoft account link

Linking your GitHub account to your Microsoft work account unlocks unlimited Copilot tokens during the hackathon. Verify it with these three commands.

If you skip this: you'll fall back to the free Copilot tier (limited completions/chat). Worth 60 seconds to confirm.

10. Three-signal validation

1. Confirm you're authenticated
gh auth status --hostname github.com

Look for ✓ Logged in to github.com as <your-username>.

2. Confirm the active username matches what you expect
gh api user --jq .login

Should print your GitHub username. If it shows someone else's, run gh auth switch.

3. Confirm the Copilot entitlement (functional check)
gh copilot explain "git status"

If you get a plain-English explanation, your entitlement is working — you're done.
If you see "Unexpected problem with GitHub Copilot in the CLI", your account isn't linked yet. Open github.com/settings/copilot and complete the link to your Microsoft work email. Then run gh auth refresh -h github.com -s copilot and retry.

Heads-up: gh copilot status is not a real subcommand — that error message is harmless, just use explain as the functional check.

Should list top-level commands.

First run prompts you to sign in with your Microsoft work account in the browser. After sign-in completes, you land at an interactive prompt — type /help to confirm skills loaded, then /exit. You're done.

Troubleshooting

  • code / gh not found after install — close and reopen your terminal so PATH refreshes.
  • winget says "no applicable installer" — your Windows is too old; update Windows or grab App Installer from the Microsoft Store.
  • gh copilot status says no access — your GitHub account isn't linked to your Microsoft work account. Visit github.com/settings/copilot.
  • Defender flags an installer — open Defender → Protection history → submit a false positive if it's one of the commands above. winget and brew installs themselves should never trigger Defender.
