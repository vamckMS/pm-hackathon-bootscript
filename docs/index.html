PM Hackathon Setup Guide

Everything you need to participate in the Microsoft PM hackathon — installed step-by-step, with a verify command after each step so you know it worked. Pick your OS and editor below; the guide will show only the steps that apply to you.

Why not just one script? Earlier versions of this repo shipped a PowerShell bootstrap. Windows Defender heuristically flags iex (irm …) download-and-execute patterns as Trojan on managed corp machines — even when the script content is clean. So we switched to plain winget / brew / apt commands you paste yourself. The old script is preserved in /legacy for advanced users on unmanaged machines.

How to use this guide: every step has a Check command first. Run it — if it prints a version, you already have that tool and can skip the install. Only run the Install command if the check fails (e.g., "command not found"). After installing, close and reopen your terminal so PATH refreshes, then re-run the check.

★ Before you start: open a terminal

All the commands below are run in a terminal / command line. Open one this way:

Windows
  • In VS Code: press Ctrl + ` (backtick, top-left of your keyboard) to open the integrated terminal. Or menu: Terminal → New Terminal.
  • Don't have VS Code installed yet? Use one of the standalone options below — you'll install VS Code later in this guide.
  • Standalone: Start menu → "Windows Terminal" (or "PowerShell" / "Command Prompt" on older Windows).
  • If you don't have Windows Terminal yet, that's fine — install it in Step 5 below. Note Step 5's check command needs PowerShell (not cmd); every other command in this guide works in either.
macOS
  • In VS Code: press Ctrl + ` (backtick) to open the integrated terminal. Or menu: Terminal → New Terminal.
  • Don't have VS Code installed yet? Use Spotlight below — you'll install VS Code later in this guide.
  • Standalone: Spotlight (Cmd + Space) → type "Terminal" → Enter. Or open Applications → Utilities → Terminal.
Linux
  • In VS Code: press Ctrl + ` (backtick) to open the integrated terminal. Or menu: Terminal → New Terminal.
  • Don't have VS Code installed yet? Use the standalone option below — you'll install VS Code later in this guide.
  • Standalone: Ctrl + Alt + T on most distros, or search for "Terminal" in your app launcher.

Tip: the integrated terminal inherits your current folder as the working directory — handy once you start cloning repos.

Since you picked the CLI-only path, every command in this guide runs in your standalone terminal — no editor needed.

0. Verify you have winget

winget ships with Windows 10 1809+ / Windows 11. On most modern corp laptops it's already there.

Check
winget --version

Expected: something like v1.7.x. If you get "command not found," install App Installer from the Microsoft Store, then close and reopen your terminal.

0. Install Homebrew

Homebrew is the de-facto macOS package manager.

Check (skip install if a version prints)
brew --version
Install if missing (run in Terminal)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Expected: Homebrew 4.x.x. On Apple Silicon (M-series) Macs, the installer prints two follow-up commands you must run before brew is on your PATH — they look like:

echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

Run those, then re-run brew --version to confirm.

4. GitHub CLI (gh) optional, useful

Cloning repos, opening PRs, browsing issues from the terminal. Not required for Copilot.

Check (skip install if a version prints)
gh --version
Install if missing
(type -p wget >/dev/null || sudo apt install -y wget) \
  && sudo mkdir -p -m 755 /etc/apt/keyrings \
  && wget -qO- https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo tee /etc/apt/keyrings/githubcli-archive-keyring.gpg > /dev/null \
  && sudo chmod go+r /etc/apt/keyrings/githubcli-archive-keyring.gpg \
  && echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null \
  && sudo apt update \
  && sudo apt install gh -y

Then sign in:

gh auth login
gh auth status

9. GitHub Copilot CLI (standalone)

The standalone copilot binary is the easiest path — it has its own auth flow and sidesteps the gh-CLI multi-account headaches. Requires Node.js (Step 2).

Check (skip install if it prints a version)
copilot --version
Install via npm (cross-platform)
npm install -g @github/copilot

Installs the copilot command globally. On Windows, if npm complains about EACCES, reopen your terminal as Administrator or fix npm's prefix per npm docs.

First-run sign-in
copilot

Launches an interactive session. On first run it prints a device code and a URL (github.com/login/device) — open it in a browser, sign in as the GitHub account that's linked to your Microsoft work account (usually <alias>_microsoft; see Step 10), and authorize. Type /exit to leave the session.

Validate — interactive mode auto-runs your prompt
copilot -i "explain git status"

Launches Copilot, auto-runs the prompt, and prints the answer in the interactive UI — this is the most reliable validation. Type /exit when done. (The non-interactive flag -p sometimes prints a blank screen depending on terminal settings, so prefer -i.)

Entitlement error? If copilot says you don't have access, you likely signed in as the wrong GitHub account on the device-code page. To force a re-sign-in:
  1. Inside an interactive session, run /logout then /login and authorize as the Microsoft-linked account.
  2. If that doesn't help, exit and clear cached auth: delete ~/.copilot/ (Linux/macOS) or %USERPROFILE%\.copilot\ (Windows), then re-run copilot.
  3. Confirm the right account at github.com/settings/copilot — see Step 10 for the multi-account caveat.

Validate your GitHub ↔ Microsoft account link

Linking your GitHub account to your Microsoft work account unlocks unlimited Copilot tokens during the hackathon.

10. Confirm your link

If Step 9's copilot -i "explain git status" returned an answer, you're already linked — skip to Step 11 (Agency). The check below is only for fixing entitlement errors.

If copilot says "no entitlement" or won't sign you in
  1. Check the browser first. Open a fresh browser window and confirm the avatar in the top-right is your Microsoft-linked GitHub account (typically <alias>_microsoft) — not your personal handle. Switch accounts if needed; the github.com/settings/copilot page below always reflects whichever account the browser is signed in as.
  2. Now open github.com/settings/copilot. The page should show Copilot active with Microsoft as the source.
  3. Re-run copilot and at the device-code page sign in as that same account.
  4. Re-test: copilot -i "explain git status".

If the entitlement page itself shows no Copilot access, your GitHub account isn't linked to your Microsoft account yet — follow the page's "Link" prompt, then retry.

Agency

Agency is the Microsoft-internal AI CLI standard from Azure Core Compute. Install via the official one-liner — Defender allow-lists this specific aka.ms URL.

11. Install Agency

Microsoft-internal AI CLI standard from Azure Core Compute.

Check (skip install if it prints a version)
agency copilot --version
Install — run in PowerShell (any version)
iex "& { $(irm aka.ms/InstallTool.ps1) } agency"

The Microsoft installer launches and walks you through sign-in. Close and reopen your terminal after install so agency lands on PATH.

12. Validate Agency

Three quick signals that Agency is installed, on PATH, and signed in.

1. Binary on PATH
agency copilot --version

Prints a version string. If "command not found", reopen your terminal; if still missing, re-run the install in Step 11.

2. Help works
agency copilot --help

Should list top-level commands.

3. Launch an interactive session
agency copilot

First run prompts you to sign in with your Microsoft work account in the browser. After sign-in completes, you land at an interactive prompt — type /help to confirm it's working, then /exit. You're done.

Troubleshooting

  • code / gh / node not found after install — close and reopen your terminal so PATH refreshes. On macOS, also run eval "$(/opt/homebrew/bin/brew shellenv)" if needed.
  • winget says "no applicable installer" — your Windows is too old; update Windows or grab App Installer from the Microsoft Store.
  • winget blocked by policy on a managed machine — your org may distribute these tools via Intune / Company Portal instead. Check Company Portal for VS Code, Git, Python, Node before troubleshooting winget.
  • copilot says no entitlement — your active GitHub account isn't the one linked to Microsoft. See Step 10.
  • Defender flags an installer — stop, don't bypass it. Verify the command came from this guide (or a trusted source), and check with your hackathon organizers / IT before continuing. The winget, brew, apt, and aka.ms/InstallTool.ps1 commands above are the only network-fetch operations this guide uses.
